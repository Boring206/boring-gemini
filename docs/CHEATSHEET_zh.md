# ⚡ Boring-Gemini 快速參考指南 (Cheatsheet)

> **「極致強大，極致簡單。」**

## 🖐️ 五大核心指令 (The 5 Commandments)

| 指令 | 英文別名 | 說明 |
| :--- | :--- | :--- |
| **啟動開發** | `boring go` | 🚀 **一條龍模式 (One Dragon)**<br>全自動流程：設定 -> 規劃 -> 開發 -> 驗證。 |
| **自動修復** | `boring fix` | 🔧 **治療者 (The Healer)**<br>自動修復程式碼風格錯誤與 Linting 問題。 |
| **健康檢查** | `boring check` | ✅ **Vibe Check**<br>全面掃描專案健康度、安全性與品質。 |
| **智能存檔** | `boring save` | 💾 **紀錄者 (The Scribe)**<br>自動生成語意化 Commit 訊息並提交變更。 |
| **互動導航** | `boring guide` | ❓ **先知 (The Oracle)**<br>互動式工具探索與使用說明。 |
| **系統優化** | `boring doctor -o`| 💎 **完美主義者 (The Perfectionist)**<br>深度清理 (VACUUM)、大腦維護與檢查點重構。 |

## 🧠 認知工具 (Deep Thinking)

當簡單工具無法解決問題時，使用深度思考功能。

*   **深度推理**: `boring fix --think` (啟用 System 2 深度思考 - **NEW!**)
*   **功能規劃**: `boring go "新增使用者認證功能"`

### 🧬 God Mode (自主模式)
*   **The Sentinel (即時副駕駛)**: `boring watch` (監控檔案變更並主動協助)
*   **The Evolver (自主進化)**: `boring evolve "修復所有測試錯誤"` (目標導向自主循環)

## 🔍 自然語言 (通用介面)

你隨時可以直接用中文說出你的需求：

```bash
boring "幫我重構使用者服務，改用依賴注入模式"
boring "找出哪裡驗證了 JWT token"
boring "解釋為什麼這個測試會失敗"
```

## ⌨️ 快捷鍵與技巧

*   `boring` (不帶參數) → **情境式導航** (Smart Menu)
*   `boring status` → 查看專案當前狀態
*   `boring reset-circuit` → 修復「卡住」的 Agent (重置斷路器)
