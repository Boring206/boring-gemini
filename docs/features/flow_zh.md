# 一條龍引擎 (One Dragon Engine) 流程圖架構

Boring V12.0.0 引入了 **「一條龍引擎」**，這是一個基於狀態機 (FlowGraph) 的工作流運行器，取代了以往的線性執行模式。

## 🐉 設計理念
「一條龍」代表的是無縫、端到端的自主開發體驗。Boring 不再只是執行孤立的任務，而是協調一組專業的 **節點 (Nodes)**，透過共享的 **FlowContext** 進行狀態傳遞。

### 🚉 工作流節點
1.  **Architect (架構師/規劃者)**：分析需求，檢查大腦中的既有模式，並生成結構化的 `task.md`。
2.  **Builder (構建者/執行者)**：執行自主代理循環 (Agent Loop) 以完成任務。
3.  **Healer (治癒者/修復者)**：如果構建失敗，Healer 會識別根本原因（例如缺失依賴）並執行修復。
    - **安全網**：Healer 會自動建立 Git 檢查點，並在修復期間強制執行 **STRICT (嚴格)** 影子模式。
4.  **Polish (修光師/審核者)**：進行最終驗證 (Vibe Check/Linting)。如果仍有細微問題，可將工作流導回 Builder 進行修正。
5.  **Evolver (進化者/智者)**：總結 Session，萃取新的「精通技能 (Mastered Skills)」，並與全域大腦同步知識。

## ⚙️ 如何啟動
只需執行統一的工作流命令：
```bash
boring flow
# 或者使用別名
boring go
```

## 🛡️ 認知安全
FlowGraph 的設計以安全為核心。透過隔離 **Healer** 節點，系統確保實驗性修復僅在嚴格監控（影子模式）下進行，並擁有保證的還原點（檢查點）。
